你的模型更新部分可以概括为：**缓冲区样本先作为“未知新类集合 NC”被收集；缓冲区满后，不直接把每个粒度球当作一个新类，而是先对缓冲区生成粒度球，再在“粒度球层面”做聚类或分组，得到若干个新类子模型，最后把这些新类子模型加入或替换到 KNNENS 的集成中。**

核心点是：**粒度球不是类别，粒度球只是类别的局部原型或局部覆盖单元。一个新类可以由多个粒度球共同表示。**

---

## 1. 为什么不能“一个粒度球 = 一个新类”

KNNENS 原文中的更新机制本身只把所有新类统一看成一个 `NC`，其目标是“检测是否出现新类”，而不是区分不同新类；论文也说明如果要区分不同新类，可以对这些可疑样本做聚类，但这不是原文 KNNENS 的范围。

因此，你现在要做的是把 KNNENS 的：

[
{l_1,l_2,\ldots,l_q,NC}
]

扩展为：

[
{l_1,l_2,\ldots,l_q,NC_1,NC_2,\ldots,NC_{\hat{\rho}},NC_{\text{unknown}}}
]

其中：

* (NC_1,\ldots,NC_{\hat{\rho}})：已经从缓冲区中区分出来的新类型；
* (NC_{\text{unknown}})：以后继续出现、仍无法归入已有类的新样本。

你的问题中提到：即使只有一个新型标签，也会生成很多个粒球。这个现象是正常的。因为 MGNR 的粒度球是为了刻画数据的多粒度分布结构，不是为了让每个球直接对应一个类别。MGNR 的思想本来就是先用粒度球表示数据，再基于粒度球构造多粒度邻域关系，从而提升 KNN 或聚类方法的效果。

所以当 (\rho=1) 时，缓冲区中生成的所有有效粒度球都应该归为同一个新类：

[
B_1,B_2,\ldots,B_m \rightarrow NC_1
]

而不是得到 (m) 个新类。

## 2. 缓冲区满后的基本流程

### 第一步：对缓冲区样本做筛选

缓冲区中的样本不一定全是真正的新类样本，可能包含一些已知类的误判样本。因此可以先记录每个样本的 `NC` 投票强度，例如：

[
s_{NC}(x)=\frac{#NC\text{ votes}}{b}
]

只保留满足以下条件的样本进入更新阶段：

[
s_{NC}(x)\geq \tau
]

其中 (\tau) 可以取 0.6、0.7 或 0.8。
低置信度样本可以暂时保留在候选缓冲区，等后续样本更多时再判断。

这样可以减少把已知类误判样本混入新类模型的风险。

---

### 第二步：对缓冲区样本生成粒度球

对高置信度的新类缓冲样本集合：

[
B={x_1,x_2,\ldots,x_n}
]

使用 MGNR 的粒度球生成方法，得到：

[
\mathcal{G}={GB_1,GB_2,\ldots,GB_m}
]

每个粒度球包含：

[
GB_u=(C_u,r_u,n_u,DM_u)
]

其中 (C_u) 是球心，(r_u) 是半径，(n_u) 是球内样本数，(DM_u) 是分布度量。

你的 readme 中的粒度球生成逻辑也是根据球内样本到球心的平均距离定义 (DM)，并在子球加权分布度量优于父球时接受划分。

这一步的作用不是直接得到类别，而是把缓冲区样本变成更稳定的局部结构表示。

---

### 第三步：在“粒度球层面”区分新类

这是模型更新部分最关键的地方。

不要对原始点直接聚类，而是对粒度球进行聚类。也就是说，把每个粒度球看成一个带权原型：

[
GB_u \rightarrow (C_u,r_u,n_u)
]

然后根据球心距离、半径重叠、自然邻居关系或密度关系构造粒度球图。

可以定义两个粒度球之间的距离为：

[
d(GB_u,GB_v)
============

\max(0,|C_u-C_v|_2-r_u-r_v)
]

如果两个球重叠或距离很近，则认为它们更可能属于同一个新类。

然后进行球级聚类：

[
\mathcal{G}
\rightarrow
{\mathcal{G}_1,\mathcal{G}*2,\ldots,\mathcal{G}*{\hat{\rho}}}
]

其中每个 (\mathcal{G}_j) 是一个新类的粒度球集合。

---

## 3. 如何利用 (\rho)

你的实验中通过参数 (\rho\in{1,2,3}) 控制新兴标签数量。这里有两种使用方式。

### 情况一：实验允许使用已知 (\rho)

如果实验设置中认为 (\rho) 是已知参数，那么可以直接令：

[
\hat{\rho}=\rho
]

然后把缓冲区生成的粒度球划分成 (\rho) 个粒度球簇。

例如：

* (\rho=1)：所有缓冲区粒度球都属于 (NC_1)；
* (\rho=2)：把粒度球划分为两个球簇，对应 (NC_1,NC_2)；
* (\rho=3)：划分为三个球簇，对应 (NC_1,NC_2,NC_3)。

这里推荐使用**加权球级聚类**，例如加权 K-means、加权谱聚类或基于球图的聚类。权重可以取球内样本数：

[
w_u=n_u
]

这样大粒度球对新类结构的影响更大，小粒度球更多起到边界补充作用。

### 情况二：真实场景中 (\rho) 不可知

如果你希望方法更符合数据流新类发现场景，则不应直接使用真实 (\rho)，而应估计：

[
\hat{\rho}
]

可以通过以下方式估计：

[
\hat{\rho}
==========

\text{粒度球图的连通分量数}
]

或者通过轮廓系数、DBI、密度峰值、自然邻居稳定性等指标自动选择新类数。

为了实验公平，建议你在论文或实验中区分两种设置：

* **Known-(\rho) setting**：使用真实新类数；
* **Unknown-(\rho) setting**：自动估计新类数。

---

## 4. 给缓冲区样本分配新类伪标签

假设球级聚类结果为：

[
\mathcal{G}_1,\mathcal{G}*2,\ldots,\mathcal{G}*{\hat{\rho}}
]

则每个粒度球簇对应一个新类：

[
\mathcal{G}_j \rightarrow NC_j
]

如果样本 (x) 属于粒度球 (GB_u)，且：

[
GB_u\in \mathcal{G}_j
]

则给样本赋伪标签：

[
\hat{y}(x)=NC_j
]

也可以给它一个软置信度：

[
P(NC_j|x)
=========

\frac{
\sum_{GB_u\in \mathcal{G}*j}
\exp\left(
-\frac{|x-C_u|^2}{r_u^2+\epsilon}
\right)
}{
\sum*{k=1}^{\hat{\rho}}
\sum_{GB_v\in \mathcal{G}_k}
\exp\left(
-\frac{|x-C_v|^2}{r_v^2+\epsilon}
\right)
}
]

这样就可以得到缓冲区样本的新类标签置信度矩阵。

这与你前面基于 AGGD 和 readme 中的思路是一致的：先构造局部结构，再生成标签置信度矩阵，最后通过约束优化得到满足候选标签约束的标签置信度。readme 中也是先按候选标签生成粒度球、构造局部标签向量，再通过二次规划得到初始标签置信度矩阵。 AGGD 的思想则是利用图结构和标签置信度联合优化，使标签消岐更稳健。

---

## 5. 更新 KNNENS 粒度球集成

原始 KNNENS 的更新方式是：缓冲区满后，用缓冲区样本训练新的 `NC` 子模型；如果原来已经有 `NC` 子模型，则用新的替换旧的，从而保持模型规模固定。

你的方法可以改成：

对每一轮集成 (i=1,\ldots,b)，原来有：

[
H_i=
{H_i^{(l_1)},H_i^{(l_2)},\ldots,H_i^{(l_q)}}
]

更新后变为：

[
H_i=
{H_i^{(l_1)},\ldots,H_i^{(l_q)},
H_i^{(NC_1)},\ldots,H_i^{(NC_{\hat{\rho}})}}
]

其中：

[
H_i^{(NC_j)}
]

由缓冲区中被分到 (NC_j) 的样本生成。

如果某个 (NC_j) 是第一次出现，则新增该类的粒度球子集成；如果该 (NC_j) 已经存在，则用当前缓冲区中更新后的样本替换旧的 (NC_j) 子集成。这样既继承了 KNNENS 的“新类样本更新”思想，也避免无限制地增加模型规模。KNNENS 原文也强调只替换新类样本而不替换已知类样本，因为没有真实标签时替换已知类样本风险较高。

---

## 6. 更新后的预测方式

更新后，测试样本的投票标签不再只是：

[
{l_1,l_2,\ldots,l_q,NC}
]

而是：

[
{l_1,l_2,\ldots,l_q,NC_1,\ldots,NC_{\hat{\rho}},NC_{\text{unknown}}}
]

对于一个新测试样本 (x)：

1. 如果落入已知类粒度球，则投已知类票；
2. 如果落入某个新类 (NC_j) 的粒度球，则投 (NC_j)；
3. 如果既不属于已知类，也不属于已经学习到的新类，则投 (NC_{\text{unknown}})，并继续进入缓冲区。

最终仍然采用 (b) 轮集成投票的众数作为预测结果。

## 建议你在论文中这样表述模型更新思想

可以写成如下思路：

> 当缓冲区达到预设容量后，首先根据 NC 投票强度筛选高置信度新类样本，以降低已知类误判样本对模型更新的影响。然后利用 MGNR 粒度球生成机制对缓冲区样本进行多粒度表示，得到一组能够刻画新类局部数据分布的粒度球。需要注意的是，粒度球并不直接等价于类别，而是类别的局部结构单元。因此，本文进一步在粒度球层面构造邻域图或相似度图，并根据新类数量参数 (\rho) 或自适应估计的新类数 (\hat{\rho}) 对粒度球进行分组。每个粒度球组对应一个新兴类别，组内所有粒度球共同表示该类的数据分布。最后，根据粒度球分组结果为缓冲区样本赋予新类伪标签，并为每个新类生成对应的粒度球子集成，将其加入 KNNENS 集成模型中；若该新类子模型已经存在，则用当前缓冲区中更新后的子模型进行替换，从而实现模型对新兴类别的持续适应。

最重要的一句话是：

**缓冲区更新时，先生成粒度球，再对粒度球进行分组；粒度球是原型，粒度球组才对应新类。**
